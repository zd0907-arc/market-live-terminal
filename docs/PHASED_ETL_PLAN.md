为了确保一年期（超数百GB）的庞大历史数据顺利融入腾讯云并展示在手机前端，我们采取稳扎稳打的“样本 -> 全局”验证策略。

## 第一阶段：POC 概念验证 (Mac 单机小批量) - 已部分完成
- [x] 获取样本 CSV 数据 (`603639.csv`)。

## 第二阶段：Windows 工程化清洗试验 (5 只股票抽样)
等您的 Windows 电脑下载完 ZIP 包后，咱们先拿 1-2 天的压缩包试水。
1. **指令联通**: 使用刚才提供的自带 `ssh` 协议连入 Windows，确保环境互通。
2. **过滤降噪测试**: 运行专门开发的支持 ZIP 解压+正则清洗的 `etl_worker_win.py`。
   - 这 7000 个乱七八糟的文件（基金、转债、期权）将在脚本中被自动摒弃，只清洗 `60`, `68`, `00`, `30` 开头的纯正大A现货股票。
3. **输出体积测试**: 我们要看看 5 只热门股票在经过聚合清洗后输出的 SQLite 文件有多大，以评估云端磁盘负载。

## 第三阶段：云端效果验证 (可视化闭环)
如果经过第二阶段的 5 只股票试水跑成功了，产生了微缩版的 `market_data_history.db`：
1. **注入**: 从 Windows -> 云端 SQLite 覆盖这个库。
2. **渲染确认**: 打开手机或网页，观察这 5 只股票过去的“散户趋势情绪图”或“主力净流出折线”是否正常渲染出完整光滑的一年折线。无断层、不锁屏。

## 第四阶段：火力全开大洗盘
如果云端效果炸裂：
1. **启动多线程/全量任务**: 在 Windows 上敲下回车键，让电脑连续跑 10 个小时甚至更久，把那几百天的庞大 ZIP 集群彻底炼成一炉仙丹。
2. **终极挂载**: 推送上云。彻底终结历史数据痛点。
