# 选股页前端验证指引（历史复盘 + V2 利通案例）

## 1. 这次前端已经补到什么程度

### 历史复盘基础能力

旧选股页右侧复盘现在已经做了两件事：

- 时间窗不再死盯今天，而是围绕当前候选/画像日展开
- 日线会优先吃正式数据目录，并在更老区间用 `local_history` 做兜底

所以现在：

- 选 `90天`，应该能看到完整约 90 天日线
- 选 `120天`，应该能看到完整约 120 天日线
- 旧数据即使没有完整挂单，日线价格和旧资金口径也应该还能展示

### V2 实验验证模式

旧选股页已经新增：

- `V2 实验验证`

并且已经接上：

- `/selection/candidates?strategy=v2`
- `/selection/profile/{symbol}?strategy=v2`
- `/selection/trade-dates?strategy=v2`

当前页面里的 `V2 实验验证` 会额外打开：

```text
replay_validation=true
```

这表示候选列表按 Layer3 回放收益排序，用来验证历史策略是否能把强票排出来。

注意：这是历史验证排序，里面用到了信号日之后的出场结果；不能直接当成实盘无未来数据排序。

## 2. 利通案例如何在页面上验证

页面：

- `http://127.0.0.1:5173/selection-research`

步骤：

1. 策略切到 `V2 实验验证`

## 4. 为什么之前利通不在 Top10 里

之前 v2 页面用的是 Layer1 当日横截面分数。

那套分数只回答：

```text
今天的资金/活跃度/结构强不强？
```

它没有把后续持有、出场、收益结果纳入排序。

所以利通当时虽然被识别为候选，但在 Layer1 横截面里分别是：

- `2026-01-14`：`#48`
- `2026-03-25`：`#62`

现在 `V2 实验验证` 页面改成按 Layer3 回放结果排序后，利通在这两个日期都是：

```text
#1
```

这里验证的是：

```text
当日候选 -> 次日入场 -> 后续出场信号 -> 实际回放收益
```

这才是前面围绕利通复盘出来的完整策略链路。

## 5. 这次顺手修的一个前端状态问题

之前切历史日期时，右侧会短暂保留旧候选的画像，导致：

- 左边日期已经变了
- 右边还是上一只股票 / 上一个信号日

现在切日期重新查候选时，会先清掉右侧旧状态，再等新候选回来，避免混看。
