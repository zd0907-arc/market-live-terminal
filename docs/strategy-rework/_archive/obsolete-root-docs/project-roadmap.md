# 选股策略重构小项目路线图

## 1. 项目目标

把当前“吸筹/启动/出货三个静态分数”升级为：

```text
盘后 L2 量化候选 → 公司研究/事件解释 → 交易状态模拟 → 参数化回测验证
```

最终支持：

- 任意历史日回放。
- 候选 TopN 解释。
- 公司/事件研究卡片。
- 模拟入场、持有、出场。
- 月度/区间批量回测。
- 参数版本对比。

## 2. 阶段拆分

### Phase 0：只读实验底座

目标：不改现有选股页主链路，先做离线实验。

交付：

- 新策略版本命名：`selection_strategy_v2_lifecycle`。
- 新实验脚本/模块，只读正式库。
- 单票回放能力，优先 `sh603629`。
- 输出 CSV/JSON，不写业务表。

验收：

- 可复现利通电子 2026-01 到 2026-04 的主要状态。
- 不影响当前前后端页面。

### Phase 1：量化候选层

目标：实现 Layer 1 候选发现。

交付：

- 衍生指标计算：资金强度、主动买卖、挂撤单、成交额异常、价格结构。
- 候选类型识别：`launch_candidate`、`event_spike_candidate`、`shakeout_repair_candidate` 等。
- 单日候选 TopN 输出。

验收：

- 输入任意历史日，可输出候选列表和量化理由。
- 候选结果可追溯到字段和公式。

### Phase 2：交易状态与单日回放

目标：实现不含 LLM 的完整模拟交易链路。

交付：

- 状态机：`watch → launching → entered → holding → exit` 等。
- 次日模拟入场。
- 每日持有/洗盘/出货判断。
- 单票交易结果：收益、最大浮盈、最大回撤、持有天数、出场原因。

验收：

- 选择某历史日后，系统能模拟每只可买候选的入场和出场。
- 利通电子案例能输出状态路径。

### Phase 3：批量回测

目标：从单日回放扩展到月度/区间回测。

交付：

- 按交易日循环执行完整链路。
- 去重和持仓冲突处理。
- 统计胜率、平均收益、累计收益、回撤、失败案例。
- 参数版本对比。

验收：

- 可跑 2026-03 全月。
- 输出结果能支撑用户判断策略是否值得实盘。

当前进度补充：

- 已有第一版区间回测。
- 已补上下一交易日执行口径。
- 已补上近似涨跌停成交约束、滑点/手续费、最大持仓数、单日最大开仓数、同票冲突去重。
- 仍未补停牌、真实撮合、分仓资金曲线、板块差异化涨跌停规则。

### Phase 4：公司研究 + 事件解释层

目标：接入候选票的公司/新闻/事件研究。

交付：

- 当时可见事件时间线。
- 公司画像、业务、利润来源、题材、估值弹性卡片。
- LLM 固定判定模板。
- 事件质量与资金一致性结论。

验收：

- 历史回放严格避免未来函数。
- 大模型输出结构化、可解释、可缓存。

当前进度补充：

- 已先落一个不依赖 LLM 的 `research card v0.1`。
- v0.1 先解决历史时点研究卡片、事件时间线、情绪摘要快照、主题标签抽取。
- 后续再在此基础上补“公司业务/利润来源/估值弹性/LLM 固定判定模板”。

### Phase 5：产品接入

目标：把成熟结果接回选股页。

交付：

- 历史回放模式。
- 批量回测模式。
- 候选详情研究卡片。
- 状态路径/交易结果可视化。

验收：

- 用户可在页面选择历史日/区间并查看结果。
- 不破坏现有选股和复盘页面。

## 3. 优先级

最高优先级：

1. 离线实验底座。
2. 不含 LLM 的量化候选 + 动态回测。
3. 利通电子单票回放。

原因：先证明量化交易链路有用，再加厚公司研究层。

## 4. 暂不做

- 不直接改实盘交易。
- 不做自动下单。
- 不让 LLM 独立决定买卖。
- 不在第一阶段重构选股前端。
- 不把未经回测的参数硬接入正式页面。
