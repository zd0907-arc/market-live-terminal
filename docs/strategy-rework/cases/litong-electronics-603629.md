## 1. 案例定位

用户目标不是追求日内交易，而是用盘后 L2 数据找到“主力资金驱动的中期波段”：

## 4. 旧策略在这个案例上的失效点

在 `selection_signal_daily` 中，利通电子关键阶段几乎没有被旧策略有效捕捉：

| 日期 | stealth_score | stealth_signal | breakout_score | confirm_signal | distribution_score | exit_signal |

| 2026-03-26 | 49.45 | 0 | 42.29 | 0 | 41.31 | 0 |
| 2026-04-24 | 27.71 | 0 | 47.10 | 0 | 39.00 | 0 |

核心原因：

1. 旧 `stealth` 把吸筹理解成“温和净流入 + 不过热”，利通电子这类强股在早期已有明显涨幅，容易被排除。
2. 旧 `breakout` 依赖 `stealth_score >= 55`，导致启动确认被吸筹分拖死。
3. 旧 `distribution` 对高位风险识别不足，尤其未充分使用 `atomic_order_daily` 的日级挂撤单质量字段。
4. 旧策略没有“持有中确认/加仓/减仓”的状态机，只能给一天的分数。

## 5. 可转化为新策略的案例假设

### 5.1 找得到

利通电子式候选应从“资金生命周期”识别，而不是只看低位温和吸筹：

- 初期可以接受 20日涨幅已经不低，但必须看到 L2 主力/超大资金持续参与。
- 需要识别“小单/表层净流出但 L2 主力承接”的背离。
- 需要识别洗盘后的资金回补：急跌/分歧后 1-3 日 L2 主力净额重新转强。

### 5.2 拿得住

持有不是靠一次启动信号，而是每天盘后更新：

- 趋势未破：价格仍在关键均线/平台上方。
- 资金未走：L2 主力净额、超大净额、OIB/CVD 没有连续恶化。
- 洗盘可忍：急跌日若后续快速修复，且挂撤单/主动买回补，则不应机械卖出。

### 5.3 出得掉

出货不是吸筹的反向，而是独立的“高位派发/主动卖压/承接失效”识别：

- 高位放量但价格不再有效上行。
- L2 主力净额/超大净额连续转负。
- `sell_pressure_ratio` 上升、`buy_support_ratio` 下降。
- `cancel_buy_amount`、`add_sell_amount`、负向 CVD/OIB 放大。
- 尾盘资金修复失败，或开盘诱多后全天卖压占优。
