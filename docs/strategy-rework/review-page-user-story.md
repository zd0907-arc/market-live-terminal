# 选股复盘页面用户 Story 与优化空间

## 当前用户 Story

用户每天晚上复盘，不是为了调策略，而是为了做第二天交易准备。

核心问题：

```text
今天有哪些票值得看？
哪些明天可以买？
哪些只是观察？
系统为什么这么判断？
买了以后什么时候该跑？
```

## 还值得优化的方向

### 1. 今日复盘摘要

页面顶部增加一条轻量摘要：

```text
今日可买 3 只，观察 12 只，风险提示 1 只。
资金流回调稳健：1 只可买。
趋势中继高质量回踩：2 只可买，10 只观察。
```

价值：用户打开页面先知道今天工作量。

### 2. 明日操作清单

把“明日可操作”进一步变成交易准备清单：

```text
股票
策略来源
买入触发日
计划执行日
核心理由
主要风险
```

价值：减少用户从列表里手动摘票。

### 3. 观察池跟踪

观察中的股票应支持跨日跟踪：

```text
首次进入观察池日期
已经观察几天
是否接近买入确认
资金是否继续留场
是否失效
```

价值：避免每天看到一堆观察票但不知道谁在变好。

### 4. 策略解释继续结构化

当前已在第一个格子做解释浮窗。下一步可以把指标解释统一成字典：

```text
字段名
中文名
算法
业务含义
达标阈值
强弱分层
适用策略
```

价值：后续每个策略都能复用，不靠前端硬写。

### 5. 持仓/卖出监控

现在页面偏选股。下一步要补“已买入后”的每日监控：

```text
当前持仓
累计超大单是否撤退
是否出现卖出信号
卖出信号原因
次日操作建议
```

价值：这和买入同等重要，尤其符合用户“跟资金跑”的核心思路。

### 6. 多策略去重与冲突处理

同一只票可能被多个策略命中。后续需要：

```text
合并同票多策略信号
显示策略共振
如果一个策略买入、另一个策略风险提示，要明确冲突
```

价值：多策略越多，冲突处理越重要。

## 建议优先级

1. 今日复盘摘要。
2. 明日操作清单。
3. 观察池跨日跟踪。
4. 持仓/卖出监控。
5. 指标解释字典化。
6. 候选票事件解释卡：把公告/问答/新闻/公司逻辑接到右侧详情。
7. 消息触发快速研判卡：从一条消息反查关联股票、事件强度、资金确认和后续跟踪动作。
8. 多策略共振/冲突处理。
