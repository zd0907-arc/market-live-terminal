# 主 Agent + 子 Agent 研究工作流

## 结论

后续主 Agent 负责方向和判断，子 Agent 负责具体实验。不要让一个 Agent 同时陷入规划、写代码、跑全量、整理文档。

## 分工

### 主 Agent

职责：

```text
和用户确认研究方向
拆分假设
定义实验口径
派发子 Agent
审阅结果
决定采纳/不采纳
维护策略族和版本命名
```

主 Agent 不应反复亲自跑所有实验，除非任务很小。

### 子 Agent

职责：

```text
读取指定背景文档
只做被分配的实验
写/改脚本
跑结果
输出 README/summary/result
给出简短结论
```

子 Agent 不负责改产品方向，不负责随意升级策略版本。

## 子 Agent 任务模板

派发任务时必须包含：

```text
业务背景：当前策略族、当前默认方法版本、用户目标
问题：这次只验证什么
输入数据：数据库、日期范围、股票范围、已有实验路径
允许改动：能改哪些脚本/只能新增哪些文件
禁止事项：不要改前端、不要覆盖旧实验、不要发明新命名
输出要求：README.md、summary.json、result.csv、必要的对比表
验收口径：看哪些指标，如何判断有效
```

## 推荐并行方向

当前可以并行拆给子 Agent 的方向：

| 方向 | 归属 | 目的 |
|---|---|---|
| 市场最强/最弱 30 反推 | `S01` / `S02` | 找当前发现层错过牛股、误入弱票的原因 |
| ST/冷却/市值/早期资金失败拆解 | `S01` | 不把业务防线混成一个 v1.5，逐条看贡献 |
| 成交额绝对门槛替代 | `S01` / `S02` | 用换手、相对成交额、个股历史分位替代 2.5 亿硬门槛 |
| 累计超大单持有/退出归因 | `S04` | 验证资金没跑就拿、资金撤退就卖的稳定性 |
| 挂单诱多模型复盘 | `S04` | 研究撤梯子、压单、挂单成交背离 |
| 消息面/公司研究原型 | `S03` | 后续接新闻和大模型解释层 |

## 子 Agent 输出的判断口径

子 Agent 最终回答只需要：

```text
做了什么
结果在哪里
关键数字
我认为是否采纳
如果不采纳，为什么
下一步建议
```
