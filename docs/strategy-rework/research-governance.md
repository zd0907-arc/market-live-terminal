# 策略研究治理规则

## 结论

后续研究不再把所有想法都混在一个版本里。每条方向要先定性为：新策略族、方法版本、实验、还是参数扫描。

## 四类变更

| 类型 | 什么时候用 | 是否新建策略族/版本 |
|---|---|---|
| 新策略族 | 目标行情变了，例如从资金流反转切到消息面重估 | 新建 `Sxx-*` |
| 方法版本 | 同一类行情，但核心判断链路变了 | 新建 `Mxx-*` |
| 实验 | 验证一个假设、归因一个问题、做样本反推 | 新建 `EXP-*` |
| 参数扫描 | 只调阈值，例如 1.5/2.0/2.5 | 放在同一实验下，不升版本 |

## 一个实验必须留痕

每个实验目录至少包含：

```text
README.md          # 人能看懂的实验说明
summary.json       # 程序可读摘要
result.csv         # 主要结果
notes.md           # 可选，人工观察/复盘记录
```

`README.md` 固定包含：

```text
1. 问题
2. 假设
3. 数据范围
4. 样本口径
5. 规则/参数
6. 输出文件
7. 核心结果
8. 结论：采纳 / 不采纳 / 继续观察
```

## 当前目录处理原则

现有目录不马上搬迁，避免破坏引用：

```text
docs/strategy-rework/strategies/v1-trend-reversal-confirmation/
```

它在新体系中等价于：

```text
S01-capital-trend-reversal
```

后续新研究优先使用新路径：

```text
docs/strategy-rework/strategies/S01-capital-trend-reversal/
docs/strategy-rework/strategies/S02-capital-breakout-continuation/
...
```

如果旧实验还要引用，文档里写映射，不强行移动历史 CSV。

## 采纳标准

策略结果不能只看一次收益。至少同时看：

```text
交易数
胜率
平均收益
中位收益
最大收益
最大亏损
收益分布
是否依赖少数极端票
是否符合业务逻辑
是否能解释失败样本
```

用户已经明确：不要只靠小阈值微调，例如 `5%` 改 `6%`。如果改动不能解释“为什么更符合 A 股资金行为”，不应升级为正式方法版本。
