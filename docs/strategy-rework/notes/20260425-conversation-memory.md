# 2026-04-25 策略重构对话短期记忆

## 用户已认可/强化的方向

1. 可以采用“资金生命周期模型”，但必须能参数化、能回测、能输出量化结果。
2. 每一种行为强弱都要有阈值，例如主力资金强弱、挂单承接强弱、洗盘修复强弱、出货强弱。
3. 策略不是写死一次，而是后续要通过回测调整参数。
4. 新闻/事件线非常重要：同样涨停，纯数字看不懂质量。
5. 需要数字信号 + 新闻上下文 + 大模型固定判定套路的综合判断。
6. 有些启动是突发事件驱动，不一定提前吸筹，不能被“必须有吸筹”规则过滤。
7. 事件驱动票可能先启动，再砸盘/洗盘，再吃货/回补，再二次拉升；利通电子中用户吃过这个亏。
8. 目标玩法是跟随游资/机构资金方向，比主力慢一点进、慢一点出，吃中间波段。
9. 目标不是日内，而是一周到一两个月的中期波段。
10. 每轮对话里有用信息必须沉淀到 `docs/strategy-rework/`，不能只停留在聊天上下文。

## 对方案的影响

- 生命周期状态机要增加“事件驱动启动”路径。
- `启动确认` 不能只从 `吸筹完成` 跃迁，也可以从 `event_spike` 跃迁。
- 回测系统必须支持参数配置、批量跑、多指标输出。
- 大模型判断要放在候选生成后，作为解释/筛选/置信度模块，而不是直接替代量化模型。

## 本轮追加：产品使用形态

用户认可“三层方案”：

1. 量化候选：先按某个历史日筛出 TopN。
2. 事件解释：大模型按当时可见新闻逐票判断事件质量。
3. 交易状态：给出进场、持有、出场，并能模拟收益。

关键补充：

- 系统要支持选择任意历史日，例如 2026-03-02。
- 当日筛出的候选要进入新闻/事件判断。
- 如果可进场，应生成下一交易日模拟入场点。
- 入场后每天盘后模拟一次持有/洗盘/出货/退出判断。
- 出场后输出确定性交易结果：入场日、出场日、收益率、持有天数、出场原因。
- 还要支持对整个月每个交易日批量跑完整链路，汇总胜率、平均收益、累计收益等。
- 历史新闻必须遵守时间点约束，不能用未来新闻做过去判断。

## 本轮追加：公司研究层

用户确认整体三层框架没问题，并进一步要求第二层不能只是新闻解释，而要像研究公司一样解释候选票。

新增关键点：

- 量化筛出候选后，需要研究这家公司本身。
- 页面上要清楚展示主营业务、利润来源、业务转型、相关板块、价格驱动因素。
- 利通电子案例中，用户能拿住是因为理解其从低利润消费电子业务转向算力租赁，并能粗算利润和估值空间。
- 大模型应帮助用户形成“为什么这票可能涨、逻辑是否成立、风险在哪里”的解释。
- 这层不一定直接决定买卖，但必须辅助用户晚间复盘和第二天盘中决策。
- 公司研究层会影响持有耐心：中期逻辑强且资金持续，可以更能拿；纯情绪一日游要快进快出。

## 本轮开发沉淀

- 已把 Layer 2 先实现为不依赖 LLM 的 `research card v0.1`。
- `research card v0.1` 当前包含：公司基础信息、事件时间线、情绪摘要快照、主题标签、基础 action hint。
- 研究卡片严格按历史时点截断：
  - `stock_events.published_at <= signal_date`
  - `sentiment_events.pub_time <= signal_date`
  - `sentiment_daily_scores.trade_date <= signal_date`
- 这只是第二层的底座，不代表最终研究深度；后面还要继续补公司主营、利润来源、估值弹性、题材映射、LLM 固定判定模板。

## 本轮方向调整

- 用户明确要求：Layer 2 暂时不要继续深挖，先把 Layer 3 做起来。
- 当前优先级切换为：先把交易状态与回测执行层立起来，让三层框架先完整跑通。
- Layer 3 当前已补第一版真实交易约束：next-open 执行、近似涨跌停约束、滑点/手续费、持仓数限制、单日开仓数限制、基础资金曲线。

## 本轮继续确认

- 用户明确说：组合资金管理、停牌约束、分批止盈减仓这些先不要继续展开。
- 当前更重要的是把“进场点 / 出场点识别”本身做好。
- 重点是用一系列组合指标识别：
  - 吸筹
  - 启动攻击
  - 暴跌洗盘
  - 真正出货
  - 暴拉后的派发
- 要把 L1 / L2 真实资金流和挂单/撤单结果放在一起评估，而不是只看单一净流入。
