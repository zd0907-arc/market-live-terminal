# 系统协同开发与部署架构白皮书 (V4.0 分布式架构)

本文档详细描述了 ZhangData 系统在引入“异地分布式爬虫 Worker”和“离线历史数据分析核心”后的全新开发与部署协作模式。

## 🌐 整体物理拓扑图

目前我们的系统跨越了三台物理/虚拟设备，各自承担独立的原子功能：

1. **Mac 电脑 (开发核心 & 调度总控)**
   - **角色**: 唯一允许修改源代码的地方。
   - **职责**: 编写前端 React/Vite 代码、编写后端 FastAPI 逻辑、维护数据库表结构（Scheme）、触发部署脚本。
   - **禁忌**: 不再负责每天挂机跑拉取数据的任务。

2. **Windows 电脑 (家里 - 数据 Worker 节点)**
   - **角色**: “数据搬运工”与“离线计算集群”。
   - **优势**: 使用家庭动态宽带，自带物理防封 IP 属性。可以挂载庞大的硬盘读取 G 级别的离线 L2 CSV 历史交易数据。
   - **职责**:
     - 跑历史：读取近一年的单笔离线大部头 CSV，生成历史资金流向表并推送到云端。
     - 跑现货：每天交易时间段内，自动执行轻量级爬虫脚本通过 AkShare 获取最新日内数据，然后通过 HTTP/SSH 推送更新大屏云端数据库。

3. **Tencent Cloud / 腾讯云 (生产级展示大屏)**
   - **角色**: “纯粹的 Web 平台”与“存储中心”。
   - **职责**: 接收来自用户的移动端/PC 端 HTTP 请求并渲染 UI。存放全局统一的 SQLite 数据库。
   - **禁忌**: 被剥夺主动向互联网（东方财富等金融网站）发送爬虫请求的权力，彻底沦为被动接收数据的“大屏”。

## 🔄 全链路协同工作流 (Mac -> Github -> Cloud <- Windows)

### 场景一：日常修改代码与发版 (Mac -> Cloud)
只要您有了新点子（比如前端界面改版、加个新接口），整个过程只发生在 Mac 和腾讯云之间：
1. **[Mac]** 在 VSCode 里写代码并使用 `npm run dev` 在本地预览。
2. **[Mac]** 开发完毕后，在终端执行：
   ```bash
   git add .
   git commit -m "update: 添加了新功能"
   git push origin main
   ```
3. **[Mac]** 运行一键发布脚本将最新代码推向云端：
   ```bash
   ./deploy_to_cloud.sh
   ```
4. **[Cloud]** 腾讯云收到命令后，自动从 Github 拉取刚才你 Push 的代码，热重载并更新网页。你的手机立刻就能看到新版。

### 场景二：洗盘历史买买买数据 (Windows -> Cloud)
由于你买了一年的底层数据包，这个历史包洗盘只需要跑**一次**。
1. **[Mac]** 帮您在 Mac 上用 Python 写好那个《清洗 L2 离散逐笔包，生成聚合 30 分钟 K 线与资金分布》的脚本，然后推送到 Github。
2. **[Windows]** 在你家里那台电脑上，把整个代码库从 Github 上 `git clone`（或 pull）下来。
3. **[Windows]** 那些庞大的 CSV 历史文件夹就放在这台 Windows 的硬盘上。
4. **[Windows]** 运行那个清洗脚本。这台超强电脑会狂读 CSV，根据 `app_config` 里的超级大单阈值，算好每一天的资金净流入，生成一份含有数百万记录的 `market_data_history.db` 增量包。
5. **[Windows]** 通过写好的网络自动同步脚本，Windows 电脑将这份清洗好的绝美数据直接上载并发射到 **[Cloud]** 的 SQLite 里，永久封存！

### 场景三：每日实盘数据获取 (Windows -> Cloud)
为了对抗东方财富风控系统：
1. **[Windows]** 每天上午 9 点开市前，Windows 电脑上的定时任务（Task Scheduler）自动启动 `run_daily_worker.py` 脚本。
2. **[Windows]** 脚本伪装成散户，使用家里的宽带平稳地从 AkShare 抓取实时数据。
3. **[Windows]** 抓到数据后，通过 HTTP POST (或者直接 SSH 写库) 发送到 **[Cloud]**。
4. **[Cloud]** 接收到 Windows 传来的数据，更新 SQLite，此时你只要拿着手机（无论在哪），打开网页，就能丝滑看到云端最新的资金流。这个过程云端没有任何被封 IP 的风险。
