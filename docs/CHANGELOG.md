# 更新日志 (Changelog)

## v3.0.9
*   **Fix**: 解决由于后台数据库数据稀疏导致前端 Recharts `XAxis` 标签坐标等比渲染失真的严重 Bug。现在后端聚合接口引入了完整的 Pandas `date_range` 零值外连接补全机制。不论是盘中 72H 还是跨度 14天，X轴刻度将永远保持绝对真实的、等距物理时间映射，拒绝图形折叠。

## v3.0.8
*   **Fix**: 修复了散户情绪趋势图在“14天”历史视图下由于时间戳解析异常导致 X 轴标签渲染为 `undefined` 的 Bug。
*   **Feature**: 优化了自选股 (Watchlist) 监控添加逻辑。现在一旦将股票加入自选，后端服务除了实时盯盘，还将立刻通过高优先级后台线程异步并发触发“历史 TICK 行情全量下载”与“云端发帖情绪全量爬取”任务，确保用户一旦关注，数据即可瞬间无缝展示。

## v3.0.7
*   **Feature**: 彻底重构散户情绪趋势图表可视化工效。废除原定晦涩的单色热度柱体与“多空比”隐性折线，全面革新为“红绿看多看空评论条数堆叠柱”叠加“综合热度流量折线”。此举实现了“真实跟组人数”与“资金动能偏好”的一眼级辨识，并全新定制了带有核心净值对撞的高精度浮窗 Tooltip。

## v3.0.6 (已合并至 3.0.5 Git Tag 历史)
*   **Fix**: 修复了旧版爬虫时间戳提取异常导致散户情绪热度趋势图 (72H) 被折叠压缩成单根柱子的 Bug。现在通过解析动态 HTML 节点 `<div class="update">` 并修正跨年处理，完全恢复了情绪波段的时间切片渲染能力。
*   **Feature**: 散户情绪 (LLM) 分析升级为高频监控模式。在原有每日 `08:30` 盘前预热的基础上，追加了盘中 (`09:00 - 15:00`) 每半小时一次的自动滚动抓取与摘要生成。

## v3.0.5
*   **Architecture**: 根治云原生长效挂机时因“次日跨零点”导致的日历缓存停滞 Bug。引入自动双保险回退机制，并在 `APScheduler` 中追加每日 `00:05` 的自我焕新定时任务，确保 7x24 小时运行永不宕机。
*   **Feature**: 历史趋势支持将当日实时生成的分钟级 TICK 动态合成至右侧 K 线末尾。
*   **Frontend**: 深度重构历史 K 线组件的 ECharts 渲染引擎。彻底移除不兼容的 `visualMap`，采用正负双重离散算法，将资金面积图置底（`z:1`）且调节透明度增加通透防遮挡。
*   **Fix**: 为 Y 轴补充基于数据最值的全局刚性 `priceRange` 对冲计算，完美解决过长平坦历史数据引发图表边界计算崩溃溢出的 Bug。
*   **UI/UX**: `HistoryView` 追加 `60日` 数据展示维度及“一键重载当前 30m”刷新接口。
*   **UI/UX (Mobile)**: 深度重构移动端适配。将所有严格定宽的双列 Grid 组件更改为按需 Flex 垂直堆叠布局。屏蔽原有占据横向空间导致图表被挤压的 `逐笔成交` 右侧边栏。
*   **Feature**: 散户情绪 (LLM) 分析升级为高频监控模式。在原有每日 `08:30` 盘前预热的基础上，追加了盘中 (`09:00 - 15:00`) 每半小时一次的自动滚动抓取与摘要生成。
*   **Fix**: 修复了旧版爬虫时间戳提取异常导致散户情绪热度趋势图 (72H) 被折叠压缩成单根柱子的 Bug。现在通过解析动态 HTML 节点 `<div class="update">` 并修正跨年处理，完全恢复了情绪波段的时间切片渲染能力。

## v3.0.4
*   **Fix**: 修复守护进程下的后端拉起时序问题，保证数据库文件与关联表会在爬虫和快照服务启动之前完成强制创建与映射。

## v3.0.3
*   **Fix**: 解决项目在部署到云端由于环境隔离造成的各种依赖缺失（`python-dotenv`, `apscheduler`, `bs4` 等）。

## v3.0.0
*   **Architecture**: **云原生架构浴火重生**。全面拥抱 Docker 部署形态，摒弃“前后台挤在本地同端口开两个屏”的简陋时代。支持一键化 `docker compose` 与独立静态 Web Server。
*   **Feature**: 前后端实现 `API_BASE_URL` 自适应动态拨链，无缝切换宿主机局域网与广域网寻址。

## v2.8.0
*   **Feature**: **资金博弈 (Funds Game) 模块全面升级**。
    *   **Backend**: 深度解析腾讯行情快照，新增 **压单 (Ask1)**、**托单 (Bid1)** 及 **Tick量 (瞬时成交)** 的实时解析与持久化存储。
    *   **Frontend**: 优化“微观听诊器”面板，支持实时显示压托单挂单量及 Tick 成交差值。
    *   **UX**: 创新实现 **持久化趋势闪烁 (Persistent Trend Flash)**。价格上涨/下跌时，背景色会保留微弱红/绿底色，解决瞬时闪烁易遗漏的问题。
    *   **Fix**: 修复 Tick 量计算逻辑，增加当日日期过滤，彻底解决因历史脏数据导致的 Tick 量异常偏大问题。
    *   **Optimization**: 优化 Monitor 数据采集逻辑，确保高频数据在任何时间段均可强制写入，保证数据连续性。

## v2.7.0
*   **Feature**: **散户情绪智能分析系统**上线。接入 LLM 生成 AI 舆情摘要，支持 14 天深度回溯与增量抓取。
*   **Feature**: 新增实战派情绪词库（吃肉、织布、核按钮等），大幅提升情绪评分准确度。
*   **Feature**: 顶部栏新增 **实时时钟 (RealTimeClock)**，替代原有的静态行情时间，提供秒级刷新。
*   **Feature**: 配置面板新增 **LLM 连通性测试** 功能，支持一键验证模型 API Key 配置是否正确。
*   **UI/UX**: 情绪模块 UI 重构（左2右1布局），新增多维度趋势图（72H/14D）切换与历史摘要查询。
*   **Config**: 全新的多标签页配置面板，支持前端热更新词库与 AI 设置。
*   **Fix**: 修复 AI 摘要功能因超时和运行时错误导致的 "Network Error"，并解决数据库表 (`sentiment_summaries`) 缺失问题。
*   **Fix**: 修正 AI 摘要生成的时间戳为本地时间 (Local Time)，解决 8 小时时差问题。
