# REQ-20260316-08-monitor-multiframe-ui-slim-pro-interaction

## 1. 基本信息
- 标题：盯盘 / 复盘历史多维 UI 瘦身与专业级交互重构
- 状态：DONE
- 负责人：Codex / 前端 AI
- 关联 Task ID：`CHG-20260316-08`
- 关联 CAP：`CAP-HISTORY-30M`, `CAP-L2-HISTORY-FOUNDATION`
- 前置依赖：`REQ-20260314-08`, `REQ-20260315-01`, `REQ-20260315-03`, `REQ-20260316-07`

## 2. 背景与目标
- 当前新版盯盘页的“历史多维”虽然已具备 `5m/30m/1h/日` 同构图表与三张副图，但页面头部信息过多、缩放交互偏重开发态、移动端 tooltip 容易遮挡，整体还不符合专业金融终端的空间效率与交互预期。
- 本卡目标是在**不改变后端接口契约**的前提下，对新版历史多维做一轮纯前端重构：
  1. 把“旧版 / 新版”入口提升到全局顶部栏；
  2. 保留页面级 `[当日分时 | 历史多维]` 双区域；
  3. 把历史多维头部收敛成单行极简控制区；
  4. 新增 `15m` 维度，并在同一控制条内完成多粒度切换、步进缩放、刷新；
  5. 重写 PC / 触控板 / 手机端的图表交互，让新版图表更像专业终端。

## 3. 方案与边界
### 3.1 本卡要做什么
- `App.tsx`：
  - 将 `[旧版 / 新版]` 切换移动到顶部导航栏（搜索框右侧、配置入口左侧）；
  - 保持旧版默认值不变；
  - 保留页面级 `[当日分时 | 历史多维]`；
  - 新版历史多维接入 `15m` 粒度。
- `HistoryMultiframeFusionView.tsx`：
  - 将图表头部改为单行极简布局：
    - 左侧：`[波段复盘] + [来源 Tag] + [说明 i] + [缺失 Tag] + [缺失详情 i]`
    - 右侧：`[5分 | 15分 | 30分 | 1小时 | 日线] + [-] [+] + [刷新]`
  - 主图 x 轴开启稀疏时间标签；
  - 保留底部 slider，并调整 inside dataZoom 行为：
    - 普通滚轮/双指上下：不再默认缩放图表；
    - `Shift + 滚轮` / 横向滚动：平移；
    - `Ctrl/Cmd + 滚轮`：缩放；
  - 新增步进式 `+ / -` 缩放，粒度随 `5m/15m/30m/1h/1d` 动态切换；
  - 手机端改为长按唤出 tooltip / 十字光标，松手即消失；
  - tooltip 改为严格表格化，保证指标名、L1、L2 与数值右对齐不乱行。

### 3.2 本卡不做什么
- 不新增后端接口，不改变 `/api/history/multiframe` 返回契约；
- 不重构旧版图表实现；
- 不修改生产数据写入或 `quality_info` 生成逻辑；
- 不在本卡处理版本发布与生产部署。

## 4. 设计冻结
### 4.1 全局信息架构
- 页面仍分为：
  - 旧版：`当日分时 / 30分钟线 / 日线`
  - 新版：`当日分时 / 历史多维`
- `[旧版 / 新版]` 不再出现在主内容区，而是上移到全局导航头部。

### 4.2 历史多维头部
- 标题固定为 `波段复盘`；
- 来源 Tag 使用当前实际 source 摘要，不再长段落说明；
- 图例说明与缺失详情都折叠进 hover 气泡；
- 缺失详情列表直接消费 `quality_info`，不新增前端复杂状态机。

### 4.3 缩放系统
- 使用固定步进窗口：
  - `1d`：约 `1月 -> 3月 -> 半年 -> 1年 -> 全部`
  - `30m/1h`：约 `1周 -> 2周 -> 1月 -> 3月 -> 半年`
  - `5m/15m`：约 `当天 -> 2天 -> 5天 -> 10天 -> 1月`
- 默认步进：
  - `1d` 默认半年；
  - `30m/1h` 默认 1 月；
  - `5m/15m` 默认 5 天。

### 4.4 触控交互
- PC：保留 slider，缩放与平移尽量不打断页面滚动；
- Mobile：普通单指上下滚动不触发 tooltip；必须长按约 `300~500ms` 才进入图表查看模式。

## 5. 验收标准（Given / When / Then）
- Given 已选中任意股票，When 查看顶部导航，Then `[旧版 / 新版]` 切换应出现在搜索框右侧、配置入口左侧，且默认仍为旧版。
- Given 新版切到“历史多维”，When 查看图表头部，Then 应只看到一行紧凑控制区，而不是多块说明卡。
- Given 粒度切到 `15m`，When 请求 `/api/history/multiframe`，Then 前端应按 `granularity=15m` 拉取并展示统一图表。
- Given 鼠标悬停到说明 `i` 图标，When tooltip 弹出，Then 可看到左右柱说明、深浅色含义、质量标记说明。
- Given 存在 `quality_info`，When 鼠标悬停到缺失详情 `i` 图标，Then 应能看到具体日期 / 时间点与问题文案列表。
- Given 在 PC 端滚轮操作，When 未按 `Ctrl/Cmd`，Then 不应再频繁触发图表缩放；When 使用 `+ / -`，Then 应按当前粒度在预设窗口间步进缩放。
- Given 在手机端单指滑动页面，When 未长按图表，Then 页面应正常滚动且不弹 tooltip；When 长按并左右拖动，Then 十字光标与 tooltip 才跟随更新。

## 6. 风险与回滚
- 风险：
  1. `inside dataZoom` 与浏览器 / 触控板默认滚动存在兼容差异；
  2. 手机端长按若处理不当，可能影响整页滚动体验；
  3. 头部信息折叠后，若 tooltip 内容排版不稳，会影响可读性。
- 回滚：
  - 本卡只改前端；若出现问题，可回退 `App.tsx` 与 `HistoryMultiframeFusionView.tsx` 到上一版本，不影响后端数据与正式库。

## 7. 实施顺序（STRICT）
1. 先创建本 REQ 卡并登记 `02_BUSINESS_DOMAIN` 拟变更点；
2. 调整顶栏与新版信息架构；
3. 重构历史多维头部与粒度切换；
4. 调整 x 轴、dataZoom、步进缩放与 tooltip；
5. 接入移动端长按交互；
6. `npm run build` 验证后回填文档与 handoff。

## 8. 结果回填
- 实际改动：
  1. `src/App.tsx` 已将 `[旧版 / 新版]` 切换移入全局头部搜索框右侧，并移除主内容区原有“页面版本”控制条；
  2. 新版历史多维已支持 `5m / 15m / 30m / 1h / 日线` 五档切换，原页面内单独“时间维度”条已并入图表头部右侧；
  3. `HistoryMultiframeFusionView.tsx` 已重构为单行极简头部：左侧收敛为标题 / 来源 / 说明气泡 / 缺失提示 / 缺失详情气泡，右侧收敛为粒度切换 / `-` / `+` / 刷新；
  4. 主图与底部轴已开启稀疏时间标签显示；tooltip 改为严格表格化，数值右对齐，不再依赖自由换行布局；
  5. `dataZoom` inside 已改为更保守的专业终端交互：`Ctrl/⌘ + 滚轮` 缩放、`Shift + 滚轮` 平移，底部 slider 保留；
  6. 已新增步进式缩放预设，并为触屏设备接入“长按唤出 tooltip / 松手隐藏”的移动端交互。
- 验证结果：
  - `npm run build` ✅
- 遗留问题：
  1. `Shift + 滚轮` / 触控板横向滚动的实际手感仍受浏览器与设备驱动差异影响，后续如用户反馈可再做兼容微调；
  2. 当前只做了构建级验证，尚未做真机移动端长按交互回归；
  3. 本卡未包含版本号更新与生产发布动作。
