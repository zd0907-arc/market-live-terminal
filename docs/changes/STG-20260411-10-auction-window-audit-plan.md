## 2. 为什么现在要做这件事
前一轮评审已经确认：
- 集合竞价对 A 股研究很重要；
- 且它不能默认并入 `09:30` 第一根连续竞价 bar。

但是当前**还不能直接拍板落库形态**，因为有个前置问题没查清：

## raw 到底给了什么？
- 有无 `09:15 ~ 09:25` 数据；
- 是 `trade/order/quote` 都有，还是只有其中一部分；
- `09:25:00` 是否能被单独识别；
- 不同股票/不同日期的覆盖是否稳定。

所以这一轮的目标不是直接设计最终 schema，而是先把 raw 审计清楚。

## 3. 这轮要回答的核心问题

### 3.1 覆盖问题
对每个样本 `symbol-day`，分别确认：
- `逐笔成交.csv` 是否包含 `09:15 ~ 09:25`；
- `逐笔委托.csv` 是否包含 `09:15 ~ 09:25`；
- `行情.csv` 是否包含 `09:15 ~ 09:25`。

### 3.2 边界问题
重点确认：
- 是否存在明确的 `09:25:00` 时间点；
- `09:25 ~ 09:30` 之间是否有数据；
- `09:30` 之后是否才进入连续竞价稳定流。

### 3.3 形态问题
把样本分成几类：
- `trade+order+quote`
- `order+quote_only`
- `quote_only`
- `partial_mixed`
- `no_pre_0930_data`

### 3.4 设计问题
基于审计结果，再决定：
1. 是否单独落 `09:25:00` bar；
2. 是否统一加 `session_phase`；
3. 是否只对部分源字段开放竞价增强。

## 4. 已新增的审计脚本
- `/Users/dong/Desktop/AIGC/market-live-terminal-data-governance/backend/scripts/audit_l2_auction_window.py`

脚本用途：
- 输入日包目录或 symbol 目录；
- 自动统计 trade/order/quote 在以下窗口的覆盖：
  - `09:15:00 ~ 09:25:00`
  - `09:25:00 ~ 09:30:00`
  - `09:30:00 ~ 11:30:00`
  - `13:00:00 ~ 15:00:00`
- 输出：
  - 每类文件的 pre-0930 行数
  - 是否有 `09:25:00`
  - 各窗口首末时间
  - 样本形态分类分布
