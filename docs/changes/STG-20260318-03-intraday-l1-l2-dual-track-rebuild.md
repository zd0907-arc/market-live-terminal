# STG-20260318-03-intraday-l1-l2-dual-track-rebuild

## 1. 基本信息
- 标题：当日分时页 L1/L2 双轨重构（主力动态 + 资金博弈，STRICT 文档先行）
- 状态：ACTIVE
- 负责人：Codex / 前后端 AI
- 关联 Task ID：`CHG-20260319-01`
- 关联 CAP：`CAP-REALTIME-FLOW`, `CAP-L2-HISTORY-FOUNDATION`
- 关联 STG：`STG-20260318-03`

## 2. 背景与目标
- 背景：
  - 当前当日分时页的“主力动态”与“资金博弈分析”仍以盘中 L1 快照/逐笔阈值推演为主，无法识别撤单欺骗、冰山拆单与盘口幻觉；
  - 当前 `SentimentTrend` 仍沿用旧 `/api/sentiment*` 路径与 3 分钟思路，不满足“盘后 L2 逐笔委托 + 逐笔成交”双轨复盘要求；
  - 生产正式 L2 历史底座已具备 `history_5m_l2/history_daily_l2` 能力，但尚未把“当日盘后自动升级为 L1/L2 双轨”落到当前当日分时页。
- 本阶段目标：
  1. 把当日分时页两块核心区域——`主力动态` 与 `资金博弈分析`——统一收敛到 **5m 标准时间轴**；
  2. 冻结 **盘中 L1 单轨 / 当天盘后自动双轨 / 历史日期双轨** 三态业务规则；
  3. 用真实 `逐笔委托.csv` 事件口径补齐 L2 `add/cancel` 因子，使资金博弈从“快照猜测”升级为“订单事件测谎”；
  4. 明确后端只下发绝对指标，前端负责 VWAP、信号引擎与双图渲染；
  5. 在文档冻结后，按 Phase 1 → 4 顺序实施，不在本轮文档阶段直接建分支或改业务代码。

## 3. 总体方案与边界
- 做什么：
  - 页面范围固定为当前当日分时页，不另起新页面；
  - 覆盖两块模块：
    1. `主力动态`
    2. `资金博弈分析`
  - 时间粒度统一固定为 `5m`；
  - 页面模式固定为：
    - `intraday_l1_only`
    - `postclose_dual_track`
    - `historical_dual_track`
  - 当天盘后若 finalized L2 已到位，页面自动从 L1 单轨切到 L1/L2 双轨，不提供手动开关；
  - 后端新增统一正式接口，统一服务两块模块，旧 `/api/sentiment*` 不再作为该页面正式主路径；
  - 后端只返回基础量价与绝对资金因子；前端负责：
    - VWAP 计算
    - 文字信号生成（`吃 / 出 / 诱空 / 诱多`）
    - 双轨布局与标题态文案。
- 不做什么：
  - 本阶段不直接改线上代码、不创建开发分支；
  - 本阶段不开放调参 UI，只预留前端常量/配置入口；
  - 本阶段不把复盘页并库一并实现，只在文档中预留后续联动；
  - 本阶段不先把 `委托类型=U` 单独拆成独立业务事件，v1 统一并入 cancel，后续再视真实样本复核。

## 4. 关键决策（必须写死）
1. **统一标准桶 = 5m**
   - 当日盘中 L1 需聚合成 5m；
   - 盘后/历史 L2 也按 5m 切片；
   - 原 3 分钟资金博弈正式退场。
2. **同页双轨，不拆产品**
   - 盘中：页面保留当前信息架构，但两块都只显示 L1 单轨；
   - 盘后/历史：在当前页面基础上扩成上下双轨，不推翻整页结构。
3. **统一接口，不再让资金博弈单独走旧情绪接口**
   - 正式主路径冻结为新统一接口（见分期 REQ 与 `03_DATA_CONTRACTS`）；
   - `/api/sentiment`、`/api/sentiment/history` 保留为旧能力/兼容链路，但不再承担当日分时页正式主数据源角色。
4. **订单事件口径 v1 冻结**
   - `逐笔委托.csv::委托类型=0` → 新增委托；
   - `逐笔委托.csv::委托类型 in {1,U}` → 撤单/改单类负向事件，v1 汇总入 cancel；
   - 若后续真实样本证明 `U` 需要独立口径，再另开 `INV-*` / `MOD-*`。
5. **信号引擎前置到前端**
   - 后端不返回 `砖头/冰火` 之类图标标签；
   - 前端按 VWAP + 波动率通道 + 成交量共振 + 金额阈值生成极简文字标签。

## 5. 分阶段实施顺序
1. **Phase 1 — 逐笔委托事件化 + 5m 因子契约补齐**
   - 冻结真实样本解释；
   - 补 `total_volume / add / cancel / l2_cvd_delta / l2_oib_delta`；
   - 明确 schema、ETL、统一 API 字段映射。
2. **Phase 2 — 统一双轨接口 + 模式切换**
   - 新接口统一服务主力动态 + 资金博弈；
   - 新增 `intraday_l1_only / postclose_dual_track / historical_dual_track` 自动模式；
   - 冻结“当天盘后 finalized L2 到位即自动双轨”的判定。
3. **Phase 3 — 主力动态双轨 UI**
   - 保留当前页结构；
   - 盘中单轨，盘后/历史上下双轨。
4. **Phase 4 — 资金博弈双轨 UI + 前端信号引擎**
   - 替换旧 `SentimentTrend`；
   - 前端计算 VWAP 与极简文字信号；
   - 冻结标题态与交互文案。

## 6. 总体验收总线（Given/When/Then，绝对时间）
- Given `2026-03-18 10:00`，When 打开当日分时页，Then 页面只显示 L1 单轨 `5m` 数据，且不触发任何撤单/隐藏单信号。
- Given `2026-03-18 22:00`，When 查看当日分时页且当天 L2 已 finalized，Then `主力动态` 与 `资金博弈分析` 两块都自动切换为 L1/L2 上下双轨。
- Given `2026-03-17 14:30`，When 查看历史日期，Then 页面两块模块都返回同源 `5m` L1/L2 对比，不允许一块走新口径、一块走旧口径。
- Given `2026-03-17 09:35` 的 `逐笔委托` 样本中存在 `委托类型=1/U`，When 聚合 5m 因子，Then `cancel_*` 指标可追溯到真实 order-event 样本，不得来自盘口快照估算。

## 7. 风险与回滚
- 风险：
  1. 若不先冻结接口与 5m 因子契约，前后端会再次出现“后端返回信号、前端又重算信号”的重复职责；
  2. 若不把盘后自动双轨写成硬规则，页面容易出现“今天盘后还是单轨、历史又双轨”的混乱心智；
  3. `委托类型=U` 当前仍是 v1 假设，需保留后续调查位。
- 回滚：
  - 本轮仅落文档，不改业务代码；
  - 如方案评审需要调整，只更新 STG/REQ/core docs，不涉及生产回滚；
  - 实施阶段若 Phase 3/4 UI 未达标，允许短期保留旧组件作为安全兜底，但数据契约仍按本 STG 执行。
