# MOD-20260321-03 复盘页操作栏重构（首页同款搜索 + 最近90天范围）

## 状态
- Done（代码已完成，本地基线已通过）

## 背景
- 正式复盘页已切到生产 `L1/L2` 历史库，但顶部操作栏仍保留早期 sandbox 风格：
  - 股票选择依赖 `datalist + 手输 symbol`
  - 开始/结束日期裸露，切股后不会自动回到最近窗口
  - 顶部堆叠大量低价值标签，信息噪音高
- 用户要求把复盘页操作体验对齐首页搜索，并把顶部操作收敛为更紧凑的两行结构。

## 本轮实现
1. 复盘页股票搜索改为首页同款交互：
   - 支持名称/代码搜索
   - debounce 候选下拉
   - 允许全市场搜索，不在输入阶段做“池内限制”
2. 日期改为范围选择器弹层：
   - 开始/结束
   - 快捷区间 `30/60/90/180/全部`
   - 首次进入与切股后默认回到该股票最近 `90` 天
3. 顶部操作栏收敛：
   - 第一行：搜索 / 日期范围 / 粒度 / 锚点开关 / 清除锚点 / 执行查询
   - 后续继续精简为单行主操作区，移除第二行说明文字
4. 删除低价值标签：
   - `数据源`
   - `接口返回条数`
   - `窗口天数/点数`
   - `股票池数量`
   - `正式库最新`
   - `元数据快照`
   - 重复股票代码展示
   - `锚点：未设置`
   - 日期按钮右侧固定文案 `最近90天`
   - 粒度区 `自动` 按钮
5. 日期与粒度联动细化：
   - 日期快捷区间新增 `10天`
   - 选定日期范围后，按默认自动逻辑直接落到推荐粒度
   - 不再暴露单独的“自动”粒度按钮
6. 头部收口：
   - 保留平台 logo / 版本
   - 去掉右上“正式复盘 / L1-L2 历史”说明
7. 继续优化复盘页可读性：
   - 复用首页横向股票信息卡片风格，展示所选股票“最新一天”的基础信息
   - 日期快捷按钮移到主操作栏外侧，便于一键切换
   - `60天 / 90天` 默认粒度直接落到 `1d`
   - 股价 K 线区域高度压缩到原先约 `60%`
   - 两个“净流入对比”子图统一改为柱状图
   - tooltip 改为顶部固定、范围内约束、内容精简，避免跟随鼠标飘出画面
8. 首页 / 复盘页股票信息卡同步治理：
   - 抽出共享组件 `StockQuoteHeroCard`
   - 首页与复盘页共用同一信息卡骨架，后续信息卡样式调整优先走共享组件
   - 复盘页选股后搜索输入仅展示股票名称，不再展示 `symbol`
9. 继续按联调反馈收口：
   - 复盘页股票信息卡上移到搜索栏正下方，恢复“名称 + 股票代码”显示，且市值固定展示（无值显示 `--`）
   - 复盘页信息卡“最新日期/价格”优先使用首页同源 quote 口径，不再跟随复盘历史表最后一日
   - 锚点相关累计图从 `4` 张收敛为 `2` 张：`L2(主力+超大)`、`L1(主力+超大)`；仅在锚点开关打开时渲染
   - 压缩主图整体高度并上移底部 dataZoom，消除图表与时间范围滑条之间的大块空白
   - 首页主卡与复盘页继续共用 `StockQuoteHeroCard`，首页同步恢复代码 badge，确保后续联动改样式只改一处
10. 视觉与交互继续校准：
   - 主图高度改为固定值，避免开启/关闭锚点时整张图突然变高或变矮
   - 净流入柱图与锚点累计图改用与历史多维一致的主力/超大配色语言
   - tooltip 改为“跟随当前数据点 + 右侧自动翻左”，避免永久挡住右上角数据
11. 首页 / 复盘页导航进一步统一：
   - 首页主容器与顶栏宽度放宽到与复盘页一致的宽屏风格
   - 复盘页将搜索框移入顶栏，放到 logo 后方，对齐首页导航结构
   - 复盘页改成“顶栏搜索 → 股票信息卡 → 日期/粒度/锚点功能区 → 图表区”的顺序
   - 顶栏右侧跳转按钮在首页显示 `复盘`，进入复盘后切换为 `首页`
12. 按最新联调意见继续对齐：
   - 首页顶栏按钮文案改为 `去复盘`，复盘页同位按钮改为 `回到首页`
   - 复盘页顶栏保留与首页一致的搜索框尺寸、位置、下拉样式及右侧配置按钮占位
   - 复盘页股票信息卡补齐首页同款 meta 行（自选 / 盯盘态 / 时钟 / 核心服务状态），避免视觉与信息层再次分叉
13. 开始切到真正共享壳层：
   - 新增共用顶栏组件 `MarketTopHeader`，首页与复盘页统一复用同一套 logo / 跳转按钮 / 搜索框 / 搜索历史 / 搜索候选 / 右侧配置位
   - 新增共用股票信息 meta 行组件 `QuoteMetaRow`，首页与复盘页统一复用“自选 + 时钟 + 核心服务状态”
   - 首页把 `盯盘` 从股票信息卡移到“当日分时 / 历史多维”同一行的小按钮操作栏，复盘页卡片不再展示盯盘按钮
   - 首页“当日分时 / 历史多维”由整块 tab 改成小按钮式操作栏，作为未来主页扩展操作位的起点
14. 共享信息卡继续重构为高密度左中右版式：
   - 左侧保留名称 / 代码 / 星标 / 当前时间 / 服务状态
   - 中间改为紧凑三列：价格（今开/昨收、最高/最低）、交易（成交量/成交额、振幅/换手率）、公司信息（总市值、非交易时段最新日期）
   - 右侧大号价格 / 涨跌额 / 涨跌幅保持不变
   - `服务` 文案替换旧 `核心服务`，并补接 `turnover_rate` 到共享卡片；最新日期只在非交易时段展示
15. 首页 / 复盘页状态联动补齐：
   - 新增 URL `?symbol=` 同步机制，首页与复盘页切换时自动带上当前股票
   - 当前页选股后会用 `history.replaceState` 回写 URL，刷新页面仍可回到同一只股票

## 不变项
- 不新增后端 API
- 不修改 `/api/review/pool`、`/api/review/data` 返回契约
- 不改图表主体与“点 K 线设锚点”的行为

## 风险 / 说明
- 本地若出现 `正式复盘接口请求失败（HTTP 404）`，这不是“没数据”：
  - `无数据` 的正式口径是 **HTTP 200 + 空数组**
  - `HTTP 404` 代表当前本地运行的后端没有提供 `/api/review/*` 路由，通常是：
    - 后端没启动
    - Vite 代理没连到后端
    - 本地后端进程还是旧版本，未包含 `review.router`
- 本轮最初误在 `main` 本地工作树上直接修改，随后已转移到分支：
  - `codex/review-toolbar-refactor`
