- Given 当前主线为 `main / v5.0.0`，When 阅读 `README / 01 / 04 / AI_QUICK_START / 当前真相母卡`，Then 不再出现互相冲突的主路径、版本、架构叙事。

### C. TODO 有效性
- Given 阅读 `07_PENDING_TODO.md`，When 查任何一条待办，Then 该项必须是当前真实未完成事项，且编号不重复、状态不过期。

### D. 契约覆盖
- Given 当前真实后端接口，When 对照 `03_DATA_CONTRACTS.md`，Then 当前对外/前端消费的关键接口不缺登记。

### E. 仓库卫生
- Given 后续 AI 或人工从仓库根目录开始，When 查入口脚本、README、主文档，Then 不会再被旧 worktree、旧路径、旧命名误导。

- 风险 1：过度“清理”会误删仍有历史价值的过程文档。
  - 约束：历史卡优先保留，只改入口与状态，不做无意义重写。
- 风险 2：核心文档大范围修订后，局部字段/接口可能漏同步。
  - 约束：每批修改后都要做一次“文档内交叉检查 + 关键代码对照”。
- 风险 3：仓库卫生清理误删仍需使用的本地脚本。
  - 约束：先标记、后确认、再删；优先修路径，不先删功能脚本。
