# REQ-20260317-09-monitor-multiframe-info-strip-polish

## 1. 基本信息
- 标题：新版历史多维外置读数条与 Y 轴布局修正
- 状态：DONE
- 负责人：Codex / 前端 AI
- 关联 Task ID：`CHG-20260317-09`
- 关联 CAP：`CAP-HISTORY-30M`, `CAP-L2-HISTORY-FOUNDATION`
- 前置依赖：`ARC-CHG-20260317-monitor-multiframe-ui-slim-pro-interaction`

## 2. 背景与目标
- `v4.2.16` 上线后，用户确认新版历史多维的外置读数条与 Y 轴还有三处体验问题：
  1. Y 轴标签移动方向做反了，应该保留左轴，只把数值放到左轴右侧；
  2. 价格 / 元数据区仍然过松，OHLC 缩写也不够直观；
  3. 图表区域普通滚轮仍容易被图表截获，影响页面上下滚动。
- 目标是做一次纯前端小修正，不改接口、不改四层图结构。

## 3. 方案与边界
- 做什么：
  - Y 轴恢复为左侧，标签内嵌到轴线右侧；
  - 外置读数条中的“价格 / 元数据”改为 3 行布局，前两行每行 2 个指标，OHLC 改完整中文；
  - 收紧读数条列宽与 nowrap 策略，避免成交额和值换行错位；
  - `inside dataZoom` 不再接管普通滚轮，默认把页面滚动优先级还给浏览器，仅保留自定义 `Ctrl/⌘ + 滚轮` 缩放与 `Shift + 滚轮` 平移。
- 不做什么：
  - 不改变后端接口；
  - 不调整新版历史多维 4 层结构；
  - 不处理 sandbox 复盘页。

## 4. 验收标准
- Given 查看新版历史多维，When 观察 y 轴，Then 左轴保留，数值标签出现在左轴内侧右边，而不是整列移到图表右边。
- Given 查看外置读数条，When 观察“价格 / 元数据”，Then 应显示“开盘价 / 最高价 / 最低价 / 收盘价 / 成交额”，且前三行排布紧凑不换行。
- Given 鼠标悬停在图表区域内，When 直接滚动滚轮，Then 页面应优先上下滚动；When `Ctrl/⌘ + 滚轮`，Then 图表才缩放。

## 5. 结果回填
- 实际改动：
  1. `HistoryMultiframeFusionView.tsx` 中 yAxis 改回左侧内嵌标签；
  2. 外置读数条中的“价格 / 元数据”重排为三行中文布局；
  3. `inside dataZoom` 关闭默认滚轮接管，普通滚轮恢复页面滚动。
- 验证结果：
  - `npm run build` ✅
- 遗留问题：
  1. 浏览器与触控板驱动差异仍可能影响 `Shift + 滚轮` 横向平移手感，但普通上下滚动已优先回到页面。
