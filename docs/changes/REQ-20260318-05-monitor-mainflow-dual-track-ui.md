# REQ-20260318-05-monitor-mainflow-dual-track-ui

## 1. 基本信息
- 标题：Phase 3｜主力动态双轨 UI（盘中单轨，盘后/历史双轨）
- 状态：ACTIVE
- 负责人：Codex / 前端 AI
- 关联 Task ID：`CHG-20260319-01`
- 关联 CAP：`CAP-REALTIME-FLOW`, `CAP-L2-HISTORY-FOUNDATION`

## 2. 背景与目标
- 当前当日分时页已经有 `主力动态 (实时)` 区域，但只承载单轨 L1 语义；
- 本阶段目标是在不推翻当前页骨架的前提下，让主力动态区完成盘中单轨 / 盘后与历史双轨的扩展。

## 3. 方案与边界
- 做什么：
  - 保留当前页结构、顶部状态与成交列表区域；
  - 只改 `主力动态` 这块卡片的图表布局与标题态；
  - 使用统一接口 `bars[]` 派生主力动态所需图层。
- 不做什么：
  - 本阶段不改资金博弈卡；
  - 本阶段不新增参数调节 UI；
  - 本阶段不重做整页导航。

## 4. UI 冻结要求
### 4.1 盘中 `intraday_l1_only`
- 仅显示一张 L1 5m 图；
- 标题保留实时语义，文案建议：`主力动态（盘中 L1 5m）`；
- 不显示 L2 说明、不显示撤单/隐藏单标签。

### 4.2 盘后/历史双轨
- 上半图：`L1 表象主力动态`
- 下半图：`L2 真实主力动态`
- 两张图必须：
  - 共享同一 5m 时间轴；
  - 共享同一 tooltip 时间定位；
  - 共享同一颜色语义（L1 浅、L2 深）；
  - 保持当前页卡片风格，不做整页翻新。

### 4.3 图层约束
- 首期只展示当前主力动态最核心的量价/绝对主力买卖结构；
- 若需累计、净流、副图增强，必须复用统一 `bars[]` 前端派生，不再反向推动接口加专用字段；
- 所有图表时间轴固定 `5m`，不得再出现旧 3 分钟分桶。

## 5. 执行步骤（按顺序）
1. 把现有主力动态区改成消费统一接口；
2. 先实现盘中单轨保守兼容；
3. 再实现盘后/历史上下双轨；
4. 联调标题态、来源标签与空态。

## 6. 验收标准（Given/When/Then，绝对时间）
- Given `2026-03-18 10:00`，When 打开当日分时页，Then 主力动态区只显示一张 L1 5m 图，不出现空的下半图占位。
- Given `2026-03-18 22:00`，When 当天 finalized L2 已到位，Then 主力动态区自动切为上下双轨，上图 L1、下图 L2，共用同一时间轴。
- Given `2026-03-17 14:00`，When 查看历史日期，Then 主力动态区进入双轨，且两张图来自同一个统一接口响应。
- Given 页面结构回归检查，When 对比实施前后，Then 除主力动态卡片扩双轨外，其他区域骨架不应被推翻。

## 7. 风险与回滚
- 风险：若 UI 为双轨重构而推翻整页布局，会把本期需求膨胀成整页重做；
- 回滚：若双轨渲染联调失败，可临时退回主力动态单轨，但数据接口与模式定义不回退。

## 8. 结果回填
- 实际改动：当前仅冻结主力动态双轨布局与标题态。
- 验证结果：盘中单轨 / 盘后历史双轨的页面行为已文档化。
- 遗留问题：是否补来源徽标、是否在主力动态区复用日线融合视图的一部分视觉语义。
