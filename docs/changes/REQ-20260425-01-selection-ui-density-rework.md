# REQ-20260425-01-selection-ui-density-rework

## 1. 基本信息
- 标题：选股研究工作台 UI 密度与右侧图表优先级改造
- 状态：DONE / MERGED_TO_MAIN
- 负责人：Codex
- 关联 Task ID：selection-ui-density-20260425
- 关联 CAP：`CAP-SELECTION-RESEARCH`、`CAP-HISTORY-30M`、`CAP-STOCK-EVENTS`
- 关联 STG：独立小需求；沿用选股研究与历史复盘现有主线

## 2. 背景与目标
当前选股研究工作台在选中左侧候选后，右侧首屏优先展示了多层数据卡片、说明文本和重复控制项，用户需要向下滚动才能看到真正用于判断的波段复盘图。目标是重排前端信息层级：顶部只放操作，第二行放全局复用股票横卡，主体区域让右侧波段复盘图尽量前置；同时修正 Top10 查询语义为“每日按策略分排序取前 10”。

核心业务场景：
1. 用户选择历史某个交易日，例如 2026-03-01，查看当天启动确认 Top10 或吸筹前置 Top10。
2. 用户点选某只候选后，需要第一眼看到：该信号日在图上的位置、后续到当前的涨跌路径，以及未来可接入的出货风险提示点。
3. 用户用于验证策略准不准，而不是频繁手动调开始/结束日期、40天/90天观察窗口、分钟粒度等细控项。

## 3. 方案与边界
### 做什么
1. 顶部吸顶操作栏重排：
   - 从“返回主页面”同一行开始承载核心操作。
   - 保留：返回、页面标题、策略选择、交易日、查询候选、刷新数据。
   - 删除永久说明文字。

2. 策略入口只保留两个：
   - `breakout`：启动确认 Top10。
   - `stealth`：吸筹前置 Top10。
   - 不提供 `distribution` 作为下拉策略；出货风险属于持仓/后续路径判断，应在图上标点或在风险模块展示。

3. 候选区日期认知修正：
   - Top10 候选区只保留当前查询日期，避免查询日、结果日、最新信号日重复造成认知噪音。
   - 日期弹窗按交易日历与评分数据可用性禁点，休市日/无评分数据日不可选。

4. 股票横卡上移：
   - 复用 `StockQuoteHeroCard`。
   - 放在顶部操作栏下方，占整行。
   - 后续若该卡统一调整，首页、复盘页、选股页应同步。

5. 当前综合判断删除：
   - 不再独立展示“当前判断”行。
   - 启动确认分、吸筹分、出货分并入左侧候选卡。

6. 右侧波段复盘控制简化：
   - 去掉选股页外层重复的粒度按钮。
   - 不再展示开始日期、结束日期输入框。
   - 不再展示“信号后40天 / 90天 / 看到现在”观察窗口。
   - 改为覆盖长度快捷选项：30天、60天、90天、120天、180天。
   - 语义为“从当前查询/信号日之前向后覆盖到现在或可用数据末端”，具体实现可先用 end=今天、start=end-N天。
   - 更细的横向查看交给图表底部 dataZoom 轴。

7. 图表主体轻量增强：
   - 十字轴联动数据条保留，但首块只保留当前坐标时间与持有收益摘要。
   - 价格 K 线新增入场/出场标记。
   - 锚点累计默认按入场日开启，L2/L1 累计净流入图紧贴 K 线下方展示。

8. 下方业务依据区只做 UI 压缩：
   - 保留原始字段与信息内容。
   - “为什么选中它 / 出货风险判断 / 事件时间线 / 事件依据 / 信息来源”不改变数据逻辑。
   - 仅减少卡片套卡片、降低垂直高度，可用紧凑网格或 tab/accordion。

9. 回测区：
   - 保持底部收起状态，不作为本轮重点。

### 不做什么
1. 不改数据库 schema；仅新增轻量查询接口并修正 Top10 查询口径，不重算历史数据。
2. 不把出货风险做成选股策略入口。
3. 不重写波段复盘图表主体。
4. 不调整左侧候选列表内容结构；左侧不是本轮痛点。
5. 不在页面保留原型说明话术，例如“当前默认按信号日前后窗口看”等。

## 4. 执行步骤（按顺序）
1. 新 worktree / 分支隔离开发：`codex/selection-ui-density`。
2. 完成本文档并确认方案边界。
3. 改造 `SelectionResearchPage`：吸顶操作栏、策略 state、横卡上移、两栏主体重排。
4. 改造 `SelectionDecisionPanel`：移除横卡大块、压缩综合判断、简化图表外层控制、删除说明话术。
5. 如需要，给 `HistoryMultiframeFusionView` 增加轻量 compact/controlled props，避免选股页重复控制。
6. 压缩依据区 UI，不改字段。
7. 运行 `npm run build`，必要时运行 `npm run check:baseline`。
8. 回填变更卡结果与相关文档。

## 5. 验收标准（Given/When/Then，绝对时间）
- Given 当前日期为 2026-04-25，用户打开 `/selection-research`。
- When 用户选择“启动确认 Top10”或“吸筹前置 Top10”、选择交易日并查询。
- Then 顶部第一行只展示操作类控件，无永久说明文案。

- Given 候选列表已有结果。
- When 用户点击任一候选股票。
- Then 股票横卡出现在页面顶部第二行，右侧主体首屏优先展示波段复盘图，而不是先展示多层说明和大面积指标卡。

- Given 用户查看候选区标题。
- When 查询日与最新信号日可能不一致。
- Then 页面能明确展示查询日、结果日、最新信号日，避免误判 Top10 来源日期。

- Given 用户在右侧图表区域操作。
- When 需要调整视野范围。
- Then 只使用 30/60/90/120/180 天快捷覆盖长度和图表底部 dataZoom；不再出现开始/结束日期输入框和信号后40/90天窗口。

- Given 页面展示策略选择。
- When 打开策略下拉。
- Then 只出现“启动确认 Top10”和“吸筹前置 Top10”，不出现“出货风险”选股入口。

## 6. 风险与回滚
- 风险：股票横卡从 `SelectionDecisionPanel` 上移后，quote/watchlist/backend 状态需要从子组件提升或拆分，可能引入状态重复。
  - 控制：优先抽小组件或保持单一数据来源，不复制业务计算。
- 风险：`HistoryMultiframeFusionView` 若强行改内部控制，可能影响首页历史多维页。
  - 控制：新增可选 props，默认行为不变；选股页只开启 compact/controlled 行为。
- 回滚：恢复 `SelectionResearchPage.tsx`、`SelectionDecisionPanel.tsx`、`HistoryMultiframeFusionView.tsx` 到本分支改造前版本即可，不涉及数据迁移。

## 7. 结果回填
- 实际改动：
  - `SelectionResearchPage`：顶部改为一行吸顶操作栏并展示版本号；策略入口限定为启动确认 / 吸筹前置；股票横卡上移到主体第二行；候选区标题只保留当前日期，候选卡去标签化，只保留确认/吸筹/出货三分与一句理由。
  - 顶部交易日控件改为弹层日历选择器，不再暴露纯数字输入；接入交易日/评分数据可用性，休市日与无评分数据日禁点，并支持点击弹窗外关闭。
  - `SelectionDecisionPanel`：移除内部股票横卡、当前判断行和复盘决策视图外框；覆盖长度 30/60/90/120/180 天合并到波段复盘标题栏；删除观察窗口、起止日期输入和说明型提示。
  - `HistoryMultiframeFusionView`：新增交易计划标记能力，在价格 K 线上展示入场箭头与出场/模拟出场标记；波段复盘顶栏新增“锚点累计”按钮，默认按交易计划入场日开启，也可手动点击 K 线重设锚点；累计图移动到K线正下方，L2在上、L1在下，每张图用主力红/绿面积 + 超大紫线表达。
  - 下方依据区保留原字段，压缩卡片样式、覆盖模块与事件列表密度。
- 验证结果：`npm run check:baseline` 通过（版本一致、后端 133 tests、前端 build）；3001 与 8000 已从本 worktree 重启；API 已验证 2026-03-02/sz002733 返回入场 2026-03-03、模拟出场 2026-03-31；候选卡已去标签化；v5.0.8 完成顶栏单行瘦身、移除锚点说明框与冗余标签后 baseline 通过。
- 遗留问题：真实出场信号已优先读取 `exit_signal`；若后续无真实信号，当前先用 20 个交易日模拟出场点占位，正式出场策略仍可继续细化。
