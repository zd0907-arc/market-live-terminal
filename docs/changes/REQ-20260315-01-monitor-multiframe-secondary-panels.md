- 标题：历史多维副图扩展：净流入对比 + 买卖力度分离（统一 L1/L2 与 主力/超大单 视角）
- 状态：ACTIVE
- 负责人：Codex / 前后端 AI
- 关联 Task ID：`CHG-20260315-01`
- 关联 CAP：`CAP-HISTORY-30M`, `CAP-L2-HISTORY-FOUNDATION`
- 前置依赖：`REQ-20260314-08`, `T-010`, `T-011`

## 2. 背景与目标
- 当前新版“历史多维”第一版只落了主图：
  - 上方价格区；
  - 下方绝对买卖对比主图（左超大、右主力，L2 包裹 L1）。
- 旧版日线仍保留两张历史副图思路：
  1. 主力净流入；
  2. 买卖力度分离。
- 你最新要求是：不要简单平移旧图，而是按新版统一叙事重做——把这两张副图也纳入 `L1 vs L2`、`主力 vs 超大单` 的同一视觉语言里，并最终扩展到 `5m / 30m / 1h / 日` 全部维度。

## 3. 范围与边界
### 3.1 本卡要做什么
- 冻结两张副图的业务语义、视觉方案、计算公式与实现顺序；
- 明确首期优先复用 `/api/history/multiframe` 现有字段；
- 把 `2026-03` 正式 L2 数据未就绪造成的实现阻塞显式记录。

### 3.2 本卡不做什么
- 本卡不直接实现前端图表；
- 本卡不新增后端接口；
- 本卡不在 `2026-03` 全月正式数据未入库前强行做视觉微调。

## 4. 产品设计冻结
### 4.1 历史多维最终图层结构
- 每个粒度（`5m / 30m / 1h / 日`）最终统一为三层：
  1. **主图**：价格 + 绝对买卖对比主图（已完成第一版）
  2. **副图 A**：净流入对比
  3. **副图 B**：买卖力度分离

### 4.2 副图 A：净流入对比（重新设计）
- 目标：回答“这一根/这一天，方向上到底是谁在净流入/净流出”。
- 坐标规则：
  - 单 Y 轴、零轴居中；
  - 正值向上 = 净流入；
  - 负值向下 = 净流出。
- X 轴组织：
  - 每个时间点仍使用左右双组：
    - 左：超大单
    - 右：主力
- 组内层级：
  - `L2` 作为底柱；
  - `L1` 作为芯柱；
  - 使用相同宽度 + `barGap='-100%'` 完全重叠。
- 数据含义：
  - 超大单净流入 = `buy - sell`
  - 主力净流入 = `buy - sell`
- 视觉原则：
  - 净流入沿用主图的实体色系；
  - 正值用买入色，负值用卖出色；
  - 超大单继续用紫/青系，主力继续用红/绿系。

### 4.3 副图 B：买卖力度分离（重新设计）
- 目标：回答“同一根/同一天里，买盘与卖盘分别占总成交的力度有多强”。
- 坐标规则：
  - 单 Y 轴、零轴居中；
  - 向上 = 买入力度；
  - 向下 = 卖出力度。
- 口径定义：
  - 买入力度 = `buy_amount / total_amount`
  - 卖出力度 = `sell_amount / total_amount`
  - 对外展示为百分比；
  - 当 `total_amount <= 0` 时返回 `null`，禁止伪画成 `0%`。
- X 轴组织：
  - 仍采用“左超大 / 右主力”；
  - 组内仍采用 `L2 底柱 + L1 芯柱`。
- 与主图区别：
  - 主图看**绝对体量**；
  - 本图看**归一化力度**，用于跨日期/跨粒度比较。

### 4.4 今日未结算行规则
- 若该行是 preview（`is_finalized=false`）：
  - 两张副图都只画 L1 芯柱；
  - L2 一律留空；
  - tooltip 明确标注：
    - `未结算`
    - `当前仅 L1 实时口径`
    - `L2 待盘后覆盖`

## 5. 计算公式冻结
### 5.1 净流入
- `l1_main_net = l1_main_buy - l1_main_sell`
- `l1_super_net = l1_super_buy - l1_super_sell`
- `l2_main_net = l2_main_buy - l2_main_sell`
- `l2_super_net = l2_super_buy - l2_super_sell`

### 5.2 买卖力度
- `l1_main_buy_ratio = l1_main_buy / total_amount`
- `l1_main_sell_ratio = l1_main_sell / total_amount`
- `l1_super_buy_ratio = l1_super_buy / total_amount`
- `l1_super_sell_ratio = l1_super_sell / total_amount`
- `l2_main_buy_ratio = l2_main_buy / total_amount`
- `l2_main_sell_ratio = l2_main_sell / total_amount`
- `l2_super_buy_ratio = l2_super_buy / total_amount`
- `l2_super_sell_ratio = l2_super_sell / total_amount`

## 6. 数据层方案
### 6.1 首期数据来源
- 首期不新增接口，直接复用：
  - `GET /api/history/multiframe`
- 原因：
  - 现有接口已返回 `open/high/low/close/total_amount`
  - 以及 `l1/l2 * main/super * buy/sell`
  - 净流入与力度都可前端派生

### 6.2 红线
- 禁止新增“伪 L2”派生；
- 禁止在 `total_amount<=0` 时把力度画成 `0%`；
- 禁止 preview 行伪装成 finalized。

### 6.3 后续可选优化（非本期）
- 若未来前端性能或 tooltip 逻辑复杂度过高，可再考虑后端补：
  - `*_net`
  - `*_buy_ratio`
  - `*_sell_ratio`
- 但在未证明前端派生不足之前，不先扩后端契约。

## 7. 实现顺序（冻结）
1. 先等待 `T-010`：`2026-03` 全月正式数据下载、清洗、入库；

3. 再实现副图 A（净流入对比）；
4. 最后实现副图 B（买卖力度分离）。

## 8. 验收标准（Given / When / Then）
- Given `2026-03-24 15:30` 某交易日已完成正式回补，When 在历史多维查看任一粒度，Then 应能在主图下继续看到“净流入对比 + 买卖力度分离”两张副图。
- Given `2026-03-24 10:30` 当日未结算，When 查看历史多维中的今天数据点，Then 两张副图都只显示 L1 芯柱，且 tooltip 明确显示“当前仅 L1 实时口径”。
- Given 某根 bar 的 `total_amount<=0`，When 渲染买卖力度分离图，Then 该点应为空，不得误画为 `0%`。

## 9. 风险与回滚
- 风险：
  1. 若在正式 `2026-03` 数据未到位前强推实现，容易因为样本太少导致默认视窗与密度判断失真；
  2. 若把“绝对值图”和“力度图”做成同一口径，会失去归一化比较的意义；
  3. 若 preview 行错误显示 L2，会误导对盘中/盘后口径的理解。
- 回滚：
  - 本卡当前仅冻结设计与顺序，不改线上行为；
  - 真正实现时仍保留旧版页面与当前主图作为安全回退。
