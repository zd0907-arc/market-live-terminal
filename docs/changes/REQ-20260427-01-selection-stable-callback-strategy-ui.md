# REQ-20260427-01-selection-stable-callback-strategy-ui

- 标题：选股页接入“资金流回调稳健策略”
- 状态：DONE（已合并 main）
- 开发结果：前端下拉框已出现“资金流回调稳健”；后端已接入 `stable_capital_callback`
- 合并状态：已合并到 `main`

## 结论

把当前已验证效果最好的策略接入选股页下拉框，替换当前无效/难理解的旧策略入口。

页面展示名称统一使用：

```text
资金流回调稳健
```

不要在页面上显示 `S01-M05`、`v1.4-balanced`、`V2实验验证` 这类内部编号。

## 背景

当前选股页已有策略下拉框，但历史策略名称和逻辑不再适合当前研究结果。经过 2026-03 ~ 2026-04 L2 完整数据回测，目前最值得投产验证的是：

```text
资金流回调稳健策略
= 资金异动发现
+ 启动后回调承接确认
+ 累计超大单持有/退出
+ 组合风险过滤
```

成熟交易回测结果：

| 口径 | 交易数 | 胜率 | 平均净收益率 | 中位净收益率 | 大亏次数 |
|---|---:|---:|---:|---:|---:|
| 不加组合风险 | 41 | 78.05% | 8.44% | 5.72% | 3 |
| 加组合风险 | 35 | 88.57% | 10.76% | 8.21% | 1 |

## 需求范围

### 1. 策略下拉框

选股页策略下拉框至少保留：

```text
资金流回调稳健
```

可选保留一个对照项：

```text
旧策略对照
```

不再把以下作为主入口：

```text
启动确认
吸筹前置
出货识别
V2实验验证
VR discovery
```

### 2. 默认策略

页面默认选择：

```text
资金流回调稳健
```

### 3. 候选列表

候选列表只展示每日 Top10。

候选卡片需要能看到：

```text
股票代码/名称
排名
策略得分或排序分
买入状态：候选 / 等回调 / 可买入 / 风险过滤
核心原因摘要
组合风险标签数量
```

### 4. 策略解释

页面或弹窗需要能说明：

```text
这不是追涨停策略；
它是等启动后的回调承接确认；
买入后主要看累计超大单是否撤退；
多个风险信号同时出现时，会过滤。
```

### 5. 回测/验证入口

保留区间回测能力：

```text
选择开始/结束日期
每日取 Top10
模拟次日开盘买入
按策略出场信号次日开盘卖出
输出每笔收益、平均收益率、胜率、中位收益率、最大亏损
```

## 后端要求

需要新增或改造一个稳定策略接口，返回字段不能只给排名，还要给业务解释。

候选输出至少包含：

```text
symbol
name
trade_date
rank
score
entry_signal_date
entry_date
risk_count
risk_labels
setup_reason
launch_reason
pullback_reason
exit_plan_summary
strategy_display_name = 资金流回调稳健

```
