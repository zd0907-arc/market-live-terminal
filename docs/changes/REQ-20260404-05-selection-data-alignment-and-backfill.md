## 2. 问题定义
- 当前“看不到数据”不是页面空壳问题，核心是**右侧复盘视图依赖的正式历史库覆盖不足**。
- 选股候选本身主要来自 `local_history`，所以 `2025-01 ~ 2026-02` 的日级研究能跑；
- 但右侧多周期复盘主要依赖：
  - `history_daily_l2`
  - `history_5m_l2`
  - `stock_universe_meta`
- 本地这三块都明显缺口，导致：
  1. 名称不全；
  2. `2026-03` 以后很多票没有正式 L2 历史；
  3. 选到票后右侧图表不够完整。

## 6.2 2026-04-11 新增真实发现（右侧空白/卡顿链路）
- 右侧“复盘决策视图”不是单一问题，实际分成两类：
  1. **真实缺口型**
     - 如 `sh600157` 在 `2026-02-20 ~ 2026-02-27` 本地/云端都只有 placeholder；
     - 这不是前端 bug，而是这段正式 L2 历史本来就没补到。
  2. **画像日期越界型**
     - 如请求 `sh603629 @ 2026-03-19` 时，旧 `get_profile(...)` 会尝试把选股特征强行重算到 `2026-03-19`；
     - 但当前 `selection_feature_daily / selection_signal_daily` 最新只到 `2026-03-12`，会导致右侧画像接口卡住。
- 已修：
  - 后端 `selection_profile` 现在会对“请求日 > 最新可用画像日”的情况自动回落到最近可用画像日；
  - 前端右侧会明确提示：
    - 画像判断按最近可用画像日展示；
    - 下方图表仍按用户当前选择时间窗看真实历史。
- 这次结论：
  - 右侧空白不应再笼统理解成“没数据”；
  - 需要明确区分：
    1. 正式历史确实没补；
    2. 画像口径日期超出现有研究快照范围。
