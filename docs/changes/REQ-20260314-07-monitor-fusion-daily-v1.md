# REQ-20260314-07-monitor-fusion-daily-v1

## 1. 基本信息
- 标题：盯盘页新版入口 + 日线融合版 V1（先做日线）
- 状态：DONE
- 负责人：Codex / 前端 AI
- 关联 Task ID：`CHG-20260314-07`
- 关联 CAP：`CAP-L2-HISTORY-FOUNDATION`, `CAP-REALTIME-FLOW`

## 2. 背景与目标
- 当前盯盘页已经收敛为“当日分时 / 30分钟线 / 日线”三模块，但你希望在不破坏旧版使用体验的前提下，先为“盘页面”开出一条新版展示分支。
- 本轮目标：
  1. 在盯盘页内新增整页级“旧版 / 新版”切换入口；
  2. 新版先只重做“日线”模块；
  3. 日线新版改为“上方价格区 + 下方左右双柱 L1/L2 对比图”的融合视图；
  4. 当日未结算时保留当天数据，但仅展示 L1 浅色芯柱，等待盘后 L2 覆盖。

## 3. 方案与边界
- 做什么：
  - 在 `src/App.tsx` 增加盯盘页版本切换状态，默认仍进入旧版；
  - 新增 `HistoryDailyFusionView.tsx` 独立组件，避免侵入旧版日线逻辑；
  - 复用现有 `/api/history_analysis` 与 `/api/history/trend?granularity=1d`，不新增后端接口；
  - 对正式 `l2_history` 日线行展示完整 L1/L2 对比；对当日 `realtime_ticks` 行只展示 L1 柱；
  - 当日分时/30 分钟线在新版模式下继续复用旧组件。
- 不做什么：
  - 本轮不删除旧版日线；
  - 不重构当日分时与 30 分钟线新版；
  - 不新增后端 schema / 路由 / 发布版本号。

## 4. 执行步骤
1. 扩展前端 `HistoryAnalysisData` 类型，接住后端已返回的 L1/L2 日线字段。
2. 在盘页面增加“旧版 / 新版”切换入口，按版本决定日线渲染组件。
3. 新建融合版日线组件，做日线归一化：
   - `l2_history` finalized 行：展示 L1 + L2；
   - `realtime_ticks` 当日未结算行：只映射为 L1；
   - 其他 fallback 行（如 `sina`）不伪造 L2。
4. 使用 ECharts 双 grid 渲染价格区与绝对资金双柱主图区。
5. 回填业务文档与 handoff 短日志。

## 5. 验收标准（Given/When/Then，绝对时间）
- Given `2026-03-14 21:30`，When 打开盯盘页并选中任意股票，Then 页面可在同一处切换“旧版 / 新版”，默认仍停留旧版。
- Given `2026-03-14 21:32`，When 切到新版并进入“日线”，Then 上方显示价格区，下方显示“左超大 / 右主力”的双柱对比，且深色 L2 包裹浅色 L1。
- Given `2026-03-14 21:35`，When 某日数据为正式 `l2_history`，Then tooltip 必须同时显示该日的 L1/L2 主力与超大买卖额。
- Given `2026-03-14 21:36`，When 某日为当天实时 `realtime_ticks` 补行，Then 图中仅出现 L1 浅色芯柱，并明确提示“今日未结算，L2 待盘后覆盖”。
- Given `2026-03-14 21:38`，When 该股票没有正式 L2 日线历史，Then 新版日线必须显示专属空态，而不是把 `sina` 或空值伪装成正式 L2 柱。

## 6. 风险与回滚
- 风险：
  1. 日线价格 OHLC 依赖 `/api/history/trend?granularity=1d` 聚合结果；若正式 5m 历史未覆盖到该股票，价格区可能退化为仅 close 线；
  2. 新版资金柱采用 barGap 叠层实现左右双组包裹，后续若继续扩展更多图层需注意系列顺序；
  3. 当前当天未结算逻辑仍基于实时 L1 口径，不应误读为正式 L2。
- 回滚：
  - 直接切回盯盘页“旧版”即可恢复原体验；
  - 如需代码回滚，仅删除新版入口和 `HistoryDailyFusionView.tsx`，旧版组件不受影响。

## 7. 结果回填
- 实际改动：
  - `src/App.tsx`：新增整页“旧版 / 新版”切换，并让新版日线走独立组件；
  - `src/components/dashboard/HistoryDailyFusionView.tsx`：新增日线融合版 V1；
  - `src/types.ts` / `src/services/stockService.ts`：补齐日线 L1/L2 字段与 `1d` 趋势请求类型。
- 验证结果：
  - 新版入口默认不影响旧版三 Tab；
  - 新版日线只消费正式 L2 历史与当天 L1 实时补行，不伪造缺失的 L2；
  - 构建验证见本卡关联 handoff 记录。
- 遗留问题：
  1. 新版下的“当日分时 / 30 分钟线”仍是旧实现复用；
  2. 当天 price 区仍受“实时日线 OHLC 缺完整盘中日K”约束，后续如要做更强的当日形态展示，可再补专用接口或前端拼接策略。
