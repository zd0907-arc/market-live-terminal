## 2. 背景与目标
- 当前首页“散户情绪”模块仍沿用旧版股吧舆情实验链路：
  - 读接口会在 `GET /api/sentiment/dashboard/{symbol}` 内尝试实时触发 LLM；
  - 前端展示“Updated”实为页面刷新时间，不是数据更新时间；
  - 趋势图将“无新增样本”补齐为 0，容易被误读成“情绪归零”；
  - 顶部核心指标仍围绕 `情绪得分 / 多空词频比`，业务解释力不足。
- 本轮重构目标不是把它做成盘中高频交易信号，而是将其纠偏为：
  - **股吧舆情 / 散户一致性观察模块**
  - 用于观察热度、拥挤度、一致性风险、叙事变化
  - 首页继续保留大模块，但先修正业务真相，再逐期升级指标和图形

## 3. 方案与边界
- 做什么：
  - 冻结模块新定位：`散户一致性观察`
  - 冻结读接口新鲜度表达与 summary 异步策略
  - 冻结首页模块的四期重构顺序：`freshness -> metric -> price-linked -> coverage`
  - 同步 core docs，避免后续边做边改再次口头漂移
- 不做什么：
  - 本卡不直接改代码 / 不直接发版
  - 首期不引入多源聚合，不立即扩到雪球/新闻/公告
  - 首期不把该模块升级为盘中正式交易信号引擎

## 4. 分期顺序（按顺序）
1. Phase 1：定位纠偏 + freshness 治理 + 读接口去同步 LLM
2. Phase 2：指标引擎重构 + 首页大模块信息架构重做
3. Phase 3：价格联动趋势图与情绪-价格观察标签
4. Phase 4：抓取覆盖池分层与数据源扩展预留

## 5. 总体验收总线（Given/When/Then，绝对时间）
- Given `2026-03-22 09:30`，When 首页打开某已覆盖股票，Then 模块显示“最新样本时间”，而不是页面刷新时间。
- Given `2026-03-22 20:30`，When 查看 `72H` 舆情趋势，Then 无新增帖子时间桶必须与真实 `0` 值桶区分，不能把 gap 误画成“情绪归零”。
- Given `2026-03-22 20:40`，When 请求 `GET /api/sentiment/dashboard/{symbol}`，Then 后端不得在读接口中同步触发 LLM。
- Given `2026-03-24 10:00`，When 首页查看新版本模块，Then 顶部主指标应围绕“热度 / 一致性 / 风险 / 最新样本时间”表达，而不是继续以“情绪得分 + 多空词频比”为主。
- Given `2026-03-27 19:00`，When 查看 `72H / 14D` 视图，Then 用户应能同时观察舆情热度与价格变化，并收到情绪-价格背离提示。

## 6. 风险与回滚
- 风险：
  - 当前读接口和前端展示耦合较深，若 Phase 1 契约未先冻结，后续前后端容易各自“半升级”。
  - 老字段 `score/status/bull_bear_ratio/risk_warning` 仍被旧前端依赖，需明确兼容周期。
  - 趋势图 gap 语义若未统一到契约层，前端可能继续把空桶回退成 0。
- 回滚：
  - 文档期无线上回滚动作；
  - 后续实施若失败，允许暂时保留旧接口字段与旧版布局，但必须保持“禁止读接口同步 LLM”这一底线不回退。
