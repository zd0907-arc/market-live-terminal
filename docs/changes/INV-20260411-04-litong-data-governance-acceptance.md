## 5. 当前已经确认的边界

## 5.1 已确认不是 bug 的边界
- `2025-01 ~ 2026-02`
  - 这段本来就没有真实挂单 raw
  - 所以不可能补出：
    - `add/cancel`
    - `oib`
- 这里应该做的是：
  - 成交级特征榨干
  - 不是伪造挂单层

## 5.2 已确认是历史链路 bug 的问题
- `2026-03+` 的原始包明明有挂单；
- 旧脚本却因为事件码只认 `0/1/U`，导致事件层被误丢；
- 这次补数已经说明：
  - raw 有
  - 脚本旧
  - 不是源缺失

## 5.3 已确认是“质量问题而非不可用”
- 最近窗口大量天数都带：
  - `quality_info = OrderID 部分缺失，L2 数值可能偏小`
- 含义：
  - 可用于结构判断；
  - 不应用作绝对法证。
