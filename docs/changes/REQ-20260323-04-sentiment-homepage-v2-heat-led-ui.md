## 2. 背景与目标
- 目标：把首页模块重构成“左图右文”的稳定观察器，而不是碎卡片拼盘。

## 3. 方案与边界
- 做什么：
  - 模块高度约为首页一屏的 `2/3`
  - 左侧：顶部信息卡 + 热度主图
  - 中间：`AI 解读（预留）`
  - 右侧：来源 Tab + 原文全文流
  - 窗口冻结为 `5D / 20D`
  - 首页移除关键词/主题词区与方向类大卡
- 不做什么：
  - 不在本期上线 AI 解读
  - 不隐藏/折叠原文全文

## 4. 验收标准
- Given `2026-03-24 20:30`，When 打开首页模块，Then 左侧主图是价格 + 事件量柱 + 相对热度线，右侧仍能看到全文事件流。
- Given `2026-03-24 20:35`，When 切换来源 Tab，Then 只影响右侧原文流，不改变左侧总体热度主图。
- Given `2026-03-24 20:40`，When 窗口从 `5D` 切到 `20D`，Then 信息卡、图、原文流全部同步换口径。

## 5. 风险与回滚
- 风险：若右侧原文流长度过长，必须靠滚动而不是截断来解决。
- 回滚：允许保留旧模块组件，直到新 UI 稳定后再删。

## 6. 实施回填
- 已落地：
  - 首页模块高度约 `2/3` 屏；
  - 左侧单信息卡 + 热度主图；
  - 中间 `AI 解读（预留）` 窄区；
  - 右侧完整原文流，支持来源 Tab 与 `最新 / 最热` 排序；
  - 关键词/主题词区与方向类 KPI 已从首页主区移除。
