# 02_BUSINESS_DOMAIN (业务与算法基石)

> **核心定位**：定义金融概念、业务模型、阈值判定以及算法公式。所有关于“大单是什么”、“为什么不合并”的业务争论都在这里盖棺定论。

## 一、 “大单”与“主力”的绝对定义

系统中所有“主力资金流入”、“超大单净买入”全部基于以下两组不可动摇的阈值界定线进行分类（暂不支持不同股票采用不同阈值）：

*   **超大单 (Super Large Order)**：单笔成交金额 $\ge$ `1,000,000` 元 (一百万)。
*   **大单 (Large Order)**：单笔成交金额 $\ge$ `200,000` 元 (二十万) 且 $<$ `1,000,000` 元。
*   **主力资金 (Main Fund)**：大单与超大单的统称。

### 【资金求和公式】
*   **超大单净流入** = `昨日以来累计所有超大单买入金额` - `昨日以来累计所有超大单卖出金额`。
*   **主力净流入** = `(大单买入 + 超大单买入)` - `(大单卖出 + 超大单卖出)`。

## 二、 拆单还原的口径妥协（关于 `groupby` 的红线）

**【PM 决策记录：历史与实时口径必须一致】**
*   我们手握真实历史 L2 逐笔记录（含 `BuyOrderID`, `SaleOrderID`），理论上可以通过 Pandas `groupby` 将一千万的隐蔽拆单完美还原为一笔超级大单。
*   但是，目前使用的白天实时接口（AkShare、各类开源接口）是没有包含 `OrderID` 的残缺明细流！
*   **红线规定**：为了防止图表由于“历史段精准，今日实时段残缺”导致的严重统计断层，**目前的算法禁止使用包含 `OrderID` 的精准还原算法**。
*   **当前实施口径**：无论是洗几年前的历史，还是洗今天的实时，只要这行成交明细的 `Volume * Price` 的 **单行乘积** $< 200,000$，它就是被界定为散户，即便它是某个大基石资金的一小部分。**这是维持模型输入一致性的战略妥协**。

## 三、 多空情绪与微观指标算法

### 1. 内外盘 (OIB - Order Imbalance)
*   外盘 (Outer Vol) = 所有标为主动买入 (`B`) 成交量的一方。
*   内盘 (Inner Vol) = 所有标为主动卖出 (`S`) 成交量的一方。
*   **OIB 算法** = `(Outer Vol - Inner Vol) / (Outer Vol + Inner Vol)`
    *   范围在 `[-1, 1]` 之间，正数代表主动买气强于主动卖气，此时是追高买入氛围。

### 2. CVD (Cumulative Volume Delta)
*   **物理意义**：累计成交量差。记录开盘至今每一笔内外盘的净差额的连续积分。
*   **公式**：$CVD_t = CVD_{t-1} + (OuterVol_t - InnerVol_t)$

### 3. 冰山压单 / 隐形补单 (Iceberg / Hidden Orders)
我们通过极短时间（3秒）内**盘口挂单总量的变化**与这期间**真实发生成交的吃单量**进行比对，若发现挂单不减反增或者消耗极慢，则判定为冰山压单。
**严格判定逻辑**：
1. `外盘增量` > 500手。（必须有显著的主动买入）
2. `隐形买方支撑量` > `外盘增量 * 0.8`。也就是说空头拼命往下砸，但买买买的总量几乎没变。
3. （计算方法：`隐形支撑量 = (当前卖一挂单 - 上次卖一挂单) + 外盘增量`）。

### 4. 虚假撤托 (Spoofing)
主力在买一挂大单诱多，当价格快跌到买一价时，瞬间撤单。
**严格判定逻辑**：
1. `内盘增量` < 100手（这段时间极少有人真卖）。
2. `买一变化量` < -1000手（但买单墙突然垮塌撤单了）。
