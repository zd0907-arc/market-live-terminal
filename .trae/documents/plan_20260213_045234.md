# Fix Plan for Sentiment Analysis Module

## Problem Analysis
1.  **React Error**: `Invalid prop %s supplied to React.Fragment`.
    *   **Cause**: In `SentimentTrend.tsx`, I used `<Tooltip content={<></>} />`. Recharts tries to inject props into the `content` element, but `React.Fragment` (`<></>`) does not accept arbitrary props.
    *   **Fix**: Replace `<></>` with `null` or a dummy component like `<div style={{display: 'none'}} />`.

2.  **No Data Displayed**: `404 Not Found` in backend logs.
    *   **Cause**: The frontend is requesting `/api/market/sentiment...`, but the backend route is mounted at `/api/sentiment...`.
    *   **Fix**: Update the fetch URLs in `SentimentTrend.tsx` to remove the `/market` segment.

## Implementation Steps
1.  **Modify `src/components/dashboard/SentimentTrend.tsx`**:
    *   Update fetch URL 1: `http://localhost:8000/api/market/sentiment/history` -> `http://localhost:8000/api/sentiment/history`
    *   Update fetch URL 2: `http://localhost:8000/api/market/sentiment` -> `http://localhost:8000/api/sentiment`
    *   Fix Tooltip content: `content={<></>}` -> `content={() => null}` (or `<div className="hidden" />`).

2.  **Verification**:
    *   Frontend should auto-reload.
    *   Data should appear in the "Funds Game" (CVD) chart.
    *   Console error should disappear.
